Modifying audio filters

int aaxMixerSetFilter(aaxConfig config, aaxFilter filter)
int aaxScenerySetFilter(aaxConfig config, aaxFilter filter)
int aaxAudioFrameSetFilter(aaxFrame frame, aaxFilter filter)
int aaxEmitterSetFilter(aaxEmitter emitter, aaxFilter filter)

Update or apply new filter parameters.

  • The config parameter is the handle as returned by aaxDriverOpen.
  • The frame parameter is the handle as returned by aaxAudioFrameCreate.
  • The emitter parameter is the handle as returned by aaxEmitterCreate.
  • The filter parameter is the handle as returned by aaxFilterCreate.
  • The filter parameter defines which filter type to alter.
    See aaxFilterType which filter types are supported.
  • When updating a registered filter it is advised to first retrieve it using one of the getter functions (see below) and altering the returned filter rather than creating a new filter and applying it. That way the unmodified filter parameters remain the same after applying.

Return AAX_TRUE if successful or AAX_FALSE otherwise.

Reasons for failure are an incorrect or already destroyed class handle or an incorrect or already destroyed filter handle. 


const aaxFilter aaxMixerGetFilter(const aaxConfig config, enum aaxFilterType type)
const aaxFilter aaxSceneryGetFilter(const aaxConfig config, enum aaxFilterType type)
const aaxFilter aaxAudioFrameGetFilter(const aaxFrame frame, enum aaxFilterType type)
const aaxFilter aaxEmitterGetFilter(const aaxEmitter emitter, enum aaxFilterType type)

Return registered or default filter parameters.

  • The config parameter is the handle as returned by aaxDriverOpen.
  • The frame parameter is the handle as returned by aaxAudioFrameCreate.
  • The emitter parameter is the handle as returned by aaxEmitterCreate.
  • The type parameter is an enumerated type aaxFilterType which defines what filter type to return.
  • After retrieving the filter it is possible to examine it's parameters using the aaxFilter support functions.

Return the requested filter or AAX_FALSE if a error occurred.

Reasons for failure are an incorrect or already destroyed class handle or an incompatible type parameter.

RingModulator Effect

The RingModulator effect multiplies the input signal by a sine wave with a specified frequency which results in detuning of the original signal.

The main purpose is being to simulate atmospheric anomalies for wireless communications. But the effect is useful for other purposes too.

AeonWave 3.7.2 (version 1.01 of the effect) adds support for a timed-transition of the effect strength.

AeonWave 3.8.2 (version 1.02 of the effect) adds support for randomness.

The RingModulator effect is supported by Emitters.
RingModulator Effect parameters
The effect uses only one effect slot.
AAX_RINGMODULATOR_EFFECT
STEREO
p0
AAX_GAIN
Gain of the ring-modulator frequency signal.
Between 0.0 and 1.0
p1
AAX_LFO_FREQUENCY
Frequency of the low-frequency oscillator.
Between 0.01 and 50 Hz.
p2
AAX_LFO_DEPTH
Depth of the low-frequency oscillator.
Between 10.0 and 10000.0
p3
AAX_LFO_OFFSET
Offset position of the low-frequency oscillator.
Between 10.0 and 10000.0
RingModulator Effect state options
Specify a waveform type to enable the dynamic effect or false to disable it.
Timed-transition is a one time action from a start value to an end value which takes (1/lfo-frequency) seconds to complete.
Envelope following calculates the current amplitude and sets the value based on that with an update speed of (1/lfo-frequency) seconds. A low value for lfo-frequency therefore means it takes longer for the envelope following to reach the next value.
AAX_RINGMODULATOR_EFFECT
Valid waveform types are:
AAX_CONSTANT_VALUE (or true)
AAX_TRIANGLE_WAVE
AAX_SINE_WAVE
AAX_SQUARE_WAVE
AAX_SAWTOOTH_WAVE
AAX_CYCLOID_WAVE
AAX_RANDOMNESS
AAX_TIMED_TRANSITION
AAX_ENVELOPE_FOLLOW
AAX_INVERSE_TRIANGLE_WAVE
AAX_INVERSE_SINE_WAVE
AAX_INVERSE_SQUARE_WAVE
AAX_INVERSE_SAWTOOTH_WAVE
AAX_INVERSE_CYCLOID_WAVE
AAX_INVERSE_TIMED_TRANSITION
AAX_INVERSE_ENVELOPE_FOLLOW

All waveforms may be combined with AAX_ENVELOPE_FOLLOW_LOG or AAX_INVERSE_ENVELOPE_FOLLOW_LOG to create exponential behaviour, e.g. AAX_ENVELOPE_FOLLOW|AAX_ENVELOPE_FOLLOW_LOG

Volume Filter

Adjusting the gain of an audio-frame or mixer object with multiple registered emitters using the volume filter will affect the volume of all registered emitters. The filter makes it possible to let the library clamp the volume between a minimum value and a maximum value.

A high gain value might cause the audio compressor to take action but a too high value might result in audio clipping which produces an unpleasant form of audio distortion.

By specifying the Response Rate parameter the filter turns into an Automatic gain control. This means the volume setting defined by the Gain parameter no longer results in a static gain value but instead the software tries to match the defined gain level by constantly increasing or decreasing the internal gain value at the rate specified by the Response Rate parameter.

AeonWave version 3.1 (and version 1.1 of the volume filter) adds support for occlusion by defining a 3D obstruction (or 3D cavity) at the same position, and with the same orientation, as the audio-frame. The obstruction/cavity could influence the gain of the emitter based on the relative position of the emitter, the audio-frame(s) and listener.

AeonWave does not define the units of measurement for size, the calculations are scale invariant. Developers have to make sure that all distances and sizes used by AeonWave use the same unit.


The Volume filter is supported by Emitters, Audio-Frames, Sensors and the Mixer.
Occlusion is only supported by Audio-frames.
Volume Filter parameters
The filter uses only one filter slot.
AAX_VOLUME_FILTER
STEREO|3D
Volume filter, supported by the Mixer, Emitters and AudioFrames.
Slot 0
p0
AAX_GAIN
Current gain multiplier. A too high value might result in audio clipping.
AAX_LINEAR, AAX_LOGARITHMIC

Between 0.0 and 10.0, the default value is 1.0
p1
AAX_MIN_GAIN
Minimum gain value, anything lower will be clamped.
AAX_LINEAR, AAX_LOGARITHMIC

Between 0.0 and 1.0, the default value is 0.0
p2
AAX_MAX_GAIN
Maximum gain value, anything higher will be clamped.
AAX_LINEAR, AAX_LOGARITHMIC

Between 0.0 and 10.0, the default value is 1.0
p3
AAX_AGC_RESPONSE_RATE
Response rate (in seconds) of the Automatic gain control when set to something other than 0.0
Between 0.0 and 10.0 seconds, the default is 0.0
Slot 1
p0
AAX_OBJECT_WIDTH
The width of the occlusion object. The width is specified in the x-direction.
p1
AAX_OBJECT_HEIGHT
The height of the occlusion object. The height is specified in the y-direction.
p2
AAX_OBJECT_DEPTH
The depth of the occlusion object. The depth is specified in the z-direction.
p3
AAX_OBJECT_DENSITY_FACTOR
The density of the occlusion object.
Between 0.0 and 1.0, where 1.0 means silence when the object is directly between the emitter and sensor.
Volume Filter state options
The filter is always enabled for stereo mode.
For occlusion it is required to explicitly set the state to true (the dimensions define an obstruction) or AAX_INVERSE (the dimensions define a hole).

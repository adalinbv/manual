BitCrusher Filter

The BitCrusher filter can be used to simulate communications line degradation. A BitCrusher is a lo-fi (low fidelity) digital audio filter, which produces a distortion by the reduction of the resolution or bandwidth of digital audio data.

AeonWave only implements resolution-reduction using this filter. But since the frequency filter is applied before the bitcrusher filter it is easy to add bandwidth too.

AeonWave does however add the option to mix with a noise signal. And since the filter is applied after the frequency filter the noise is unaffected by any frequency filter settings.

If the combined Offset parameter and Depth parameter exceeds 1.0 then the depth of the low frequency oscillator is automatically clamped.

AeonWave 3.7.2 (version 1.01 of the filter) adds support for a timed-transition of the filter strength.

AeonWave 3.8.2 (version 1.02 of the effect) adds support for randomness.

The BitCrusher filter is supported by Emitters, Audio-Frames and the Mixer.
BitCrusher Filter parameters
The filter uses only one filter slot.
AAX_BITCRISHER_FILTER
STEREO
p0
AAX_NOISE_LEVEL
Level of the added white-noise.
Between 0.0 and 1.0, 0.0 equals to no added noise.
p1
AAX_LFO_FREQUENCY
Frequency of the low-frequency oscillator.
Response rate in Hertz (1/seconds) when using Envelope Following mode.
Between 0.01 and 50 Hz.
p2
AAX_LFO_DEPTH
Depth range of the low-frequency oscillator.
Between 0.0 and 1.0.
p3
AAX_LFO_OFFSET
Minimum offset value of the low-frequency oscillator.
Between 0.0 and 1.0
BitCrusher Filter state options
Specify a waveform type or envelope-following to enable the dynamic filter or AAX_FALSE to disable it.
Timed-transition is a one time action from a start value to an end value which takes (1/lfo-frequency) seconds to complete.
Envelope following calculates the current amplitude and sets the value based on that with an update speed of (1/lfo-frequency) seconds. A low value for lfo-frequency therefore means it takes longer for the envelope following to reach the next value.

Modifying audio effects

bool aaxMixerSetEffect(aaxConfig config, aaxEffect effect)
bool aaxScenerySetEffect(aaxConfig config, aaxEffect effect)
bool aaxAudioFrameSetEffect(aaxFrame frame, aaxEffect effect)
bool aaxEmitterSetEffect(aaxEmitter emitter, aaxEffect effect)

Update or apply new effect parameters.

  • The config parameter is the handle as returned by aaxDriverOpen.
  • The frame parameter is the handle as returned by aaxAudioFrameCreate.
  • The emitter parameter is the handle as returned by aaxEmitterCreate.
  • The effect parameter is the handle as returned by aaxEffectCreate.
  • The effect parameter defines which effect type to alter.
    See aaxEffectType which effect types are supported.
  • Only one delay effect type (phasing, chorus or flanging) can be assigned.
    Adding a new delay effect will overwrite the current one.
  • When updating a registered effect it is advised to first retrieve it using one of the getter functions (see below) and altering the returned effect rather than creating a new effect and applying it. That way the unmodified effect parameters remain the same after applying.

Return true if successful or false otherwise.

Reasons for failure are an incorrect or already destroyed class handle or an incorrect or already destroyed effect handle. 


const aaxEffect aaxMixerGetEffect(const aaxConfig config, enum aaxEffectType type)
const aaxEffect aaxSceneryGetEffect(const aaxConfig config, enum aaxEffectType type)
const aaxEffect aaxAudioFrameGetEffect(const aaxFrame frame, enum aaxEffectType type)
const aaxEffect aaxEmitterGetEffect(const aaxEmitter emitter, enum aaxEffectType type)

Return registered or default effect parameters.

  • The config parameter is the handle as returned by aaxDriverOpen.
  • The frame parameter is the handle as returned by aaxAudioFrameCreate.
  • The emitter parameter is the handle as returned by aaxEmitterCreate.
  • The type parameter is an enumerated type aaxEffectType which defines what effect type to return.
  • After retrieving the effect it is possible to examine it's parameters using the aaxEffect support functions.

Return the requested effect or false if a error occurred.

Reasons for failure are an incorrect or already destroyed class handle or an incompatible type parameter.

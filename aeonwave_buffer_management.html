Class AeonWave

Header file: <aax/aeonwave>
Namespace: aeonwave

 

For AeonWave Buffer objects may be shared across multiple emitters. The Buffer management code of the AeonWave class makes sure only one copy of a Buffer with a particular reference name is resident. When the reference name is a file location (either local or on a web server) it's data will be loaded and the buffer is ready for use after the request. Files should be in one of the audio formats supported by AeonWave. If the reference name does not locate a file it is up to the caller to fill the buffer with audio data afterwards.

When a buffer is not required anymore the caller still should call the destroy function to decrement the reference counter. 


   aeonwave::AeonWave aax(AAX_MODE_WRITE_STEREO);

   aeonwave::Buffer& buffer1 = aax.buffer("test.wav");
   aeonwave::Buffer& buffer2 = aax.buffer("test.wav");

   aeonwave::Emitter emitter1(AAX_ABSOLUTE);
   aeonwave::Emitter emitter2(AAX_ABSOLUTE);

   emitter1.add(buffer1);
   emitter2.add(buffer2);

   /*** Both emitters now use the same Buffer object */

   emitter1.remove_buffer();
   aax.destroy(buffer1);

   emitter2.remove_buffer();
   aax.destroy(buffer2);

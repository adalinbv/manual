Change the Playback state

int aaxEmitterSetState(aaxEmitter emitter, enum aaxState state)

These functions change the respective runtime playback state.

  • The emitter parameter is the handle as returned by aaxEmitterCreate.
  • The state parameter is an enumerated type aaxState and specifies the desired state for a class.

Return AAX_TRUE if successful or AAX_FALSE otherwise.

Reasons for failure are an incorrect or already destroyed class handle, an incompatible state value, failure to initialize using the requested setup parameters or the inability to start the mixer or capture thread. 


int aaxEmitterGetState(const aaxEmitter emitter)

  • The emitter parameter is the handle as returned by aaxEmitterCreate.

Return the current state of the class as an enumerated type aaxState or AAX_FALSE if a error occurred.

Reasons for failure are an incorrect or already destroyed class handle.

Supported values for the state parameter
AAX_INITIALIZED
Emitters can use this state to rewind the playback position.
AAX_PLAYING
Start or restart playback of the emitter.
AAX_PROCESSED
Immediately and permanently stop playback of emitters.
AAX_STOPPED
Request stopping playback of an emitter.
Based on whether the distance mode includes support for distance delay or not the library might decide to postpone actual stopping of playback. Therefore it is best to test for AAX_PROCESSED when waiting for an emitter to stop playback.
The setup will not change but the configuration is set to initialized.
AAX_SUSPENDED
Temporarily postpone (pause) playback of the emitter.
The setup and configuration will not change.
AAX_UPDATE
Force recalculation of the distance parameters if distance delay was enabled.
This is useful for sudden position jumps.

Change the Playback state

bool aaxMixerSetState(aaxConfig config, enum aaxState state)

These functions change the respective runtime playback state.

  • The config parameter is the handle as returned by aaxOpenDevice.
  • The state parameter is an enumerated type aaxState and specifies the desired state for a class.

Return true if successful or false otherwise.

Reasons for failure are an incorrect or already destroyed class handle, an incompatible state value, failure to initialize using the requested setup parameters or the inability to start the mixer or capture thread. 


int aaxMixerGetState(aaxConfig config)
  • The config parameter is the handle as returned by aaxOpenDevice.

Return the current state of the class as an enumerated type aaxState or false if a error occurred.

Reasons for failure are an incorrect or already destroyed class handle.

Supported values for the state parameter
AAX_INITIALIZED
The mixer should be initialized after the mixer has been set-up properly.
AAX_PLAYING
Start or restart playback of the mixer
AAX_STOPPED
Request stopping playback of all emitters.
The setup will not change but the configuration is set to initialized.
AAX_SUSPENDED
Temporarily postpone playback of the mixer.
The setup and configuration will not change.
AAX_STANDBY
Temporarily switch to low energy mode.
The standby mode is like AAX_PLAYING although state changes will be processed at a slower rate. The mixer is disabled and no sound will be audible.
AAX_UPDATE
Force the mixer to render the next time-frame, especially useful for batch-mode.
The size of the time-frame is refresh-rate dependent.
The resulting buffer will be pushed to the buffer queue.

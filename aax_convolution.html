Convolution Effect

Convolution is an effect that uses an impulse response recording to give a close representation of the recorded environment.

This is a resource heavy effect, use with care.

The effect has two distinct audio paths which get mixed in the end:
  • A convoluted path which represents the room response of the original signal.
  • A direct path of the original signal which can be filtered using the frequency filter.
By filtering the direct path it is possible to simulate an object or objects standing between the emitter and the listener.

There are several ways to decrease resource usage for this effect:
  • Use short impulse response files.
    Since it is possible to combine convolution with the reverb effect it would be an option to increase reverberation decay times by combining both effects.
  • Increase the threshold-level.
    Silence is handled efficiently by the effect so by increasing the silence threshold a longer impulse response file can be handled more efficiently. Setting the level too high might introduce artefacts though.
  • Decrease the mixer frequency.
    The mixer frequency influences how many samples need to be processed for every refresh period. Setting the mixer frequency to 22050Hz from 48000Hz for lower end machines will more than half the number of samples that need to be processed.

Always make sure test this effect with all used impulse response files on the lowest supported hardware for your project.

The Convolution effect is supported by Mixer.
Convolution parameters
The effect uses only one effect slot.
AAX_CONVOLUTION_EFFECT
STEREO
Slot 0
p0
AAX_CUTOFF_FREQUENCY
Alters the frequency response of the original signal by setting a Low Pass frequency filter cut off frequency.
Between 50Hz and 22kHz.
p1
AAX_LF_GAIN
Gain of all frequencies below the cut-off frequency of the original signal.
AAX_LINEAR, AAX_LOGARITHMIC

Between 0.0 and 1.0
p2
AAX_MAX_GAIN
Sets the gain for the convoluted signal.
AAX_LINEAR, AAX_LOGARITHMIC

Between 0.0 and 1.0
p3
AAX_THRESHOLD
Set the threshold level that represents silence.
AAX_LINEAR, AAX_LOGARITHMIC

Any audio value lower than the threshold will be considered silence.
Between 0.0 and 1.0
Convolution state options
Specify AAX_TRUE to enable convolution or AAX_FALSE to disable it.

Pitch Effect

Adjusting the pitch of an audio-frame or mixer object with multiple registered emitters using the pitch effect will affect the pitch of all registered emitters. The effect makes it possible to let the library clamp the pitch to a maximum value.

The hard maximum pitch-limit is 2.0 and cannot be altered.

Starting with AeonWave 3.6 (pitch-effect version 1.01) it is possible to define a starting pitch value and the amount of pitch-change per second to make the effect glide towards the value specified by AAX_PITCH automatically.

The Pitch effect is supported by the Mixer, Emitters and Audio-Frames.
Pitch Effect parameters
The effect uses only one effect slot.
AAX_PITCH_EFFECT
STEREO
p0
AAX_PITCH
Current pitch multiplier.
Between 0.0 and 2.0, defaults to 1.0
p1
AAX_MAX_PITCH
Maximum pitch value, anything higher will be saturated.
Between 0.0 and 2.0, defaults to 2.0
p2
AAX_PITCH_START
Starting value of the pitch-glide.
The end value will be the value defined by AAX_PITCH.
Between 0.0 and 2.0, defaults to 1.0
p3
AAX_PITCH_RATE
Pitch-change per second. If set to 0.5 the glide will take half a second.
The pitch will glide from AAX_PITCH_START to AAX_PITCH at AAX_PITCH_RATE.
When AAX_PITCH is reached it will remain at that value.
A positive value, defaults to 0.0 (disabled).
Pitch Effect state options
The effect is always enabled.
To enable pitch-gliding it is required to set the state to true or AAX_TIMED_TRANSITION after setting the pitch-glide parameters.
Adding AAX_ENVELOPE_FOLLOW to the state will make the pitch-glide logarithmic.

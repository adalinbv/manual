Phasing Effect

The Phasing effect adds a slightly delayed copy of the original signal to the undelayed signal.
This causes natural frequency filtering with a series of peaks and troughs which alters the Timbre of the sound.

Only one delay effect can be used at an object (mixer, emitter, audio-frame) at any given time: either Phasing, Chorus, Flanging.

In addition to following a continuous Low Frequency Oscillator level for effect strength it is also possible to alter the strength of the effect by following the level of the input signal. This makes it possible to have a strong effect for high input signal levels and no effect for low input levels or vice versa.

If the combined Offset parameter and Depth parameter exceeds 1.0 then the depth of the low frequency oscillator is automatically clamped.

AeonWave 3.7.2 (version 1.01 of the effect) adds support for a timed-transition of the effect strength.

AeonWave 3.8.2 (version 1.02 of the effect) adds support for randomness.

AeonWave 3.12.4 (version 1.04 of the effect) adds support for negative (phase shifted) levels and a delay level higher than 1.0 where the delayed signal is louder than the direct signal.

AeonWave 3.12.4 allows assigning both the delay effect and either phasing, chorus or flanging simultaneously.

The Phasing effect is supported by the Mixer, Emitters and Audio-Frames.

Phasing Effect state options
Specify a waveform type or envelope-following to enable the phasing effect or AAX_FALSE to disable it.
Timed-transition is a one time action from a start value to an end value which takes (1/lfo-frequency) seconds to complete.
Envelope following calculates the current amplitude and sets the value based on that with an update speed of (1/lfo-frequency) seconds. A low value for lfo-frequency therefore means it takes longer for the envelope following to reach the next value.
